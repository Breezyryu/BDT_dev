# PNE .sch 바이너리 필드 맵 교차 검증 결과

> 분석일: 2025-07-14  
> 대상: Rawdata/ 전체 122개 .sch 파일 (44개 고유 스케줄)  
> 검증 스텝: CHG_CC 157개, CHG_CCCV 84개, DCHG_CC 253개, REST 278개, LOOP 123개 = **총 895개 블록**

## 1. 파일 전체 구조

```
┌──────────────────────────────────────────────────┐
│  Offset 0~15    : Magic Header (4 × uint32)       │
│  (740721, 131077, 50, 0) — 전 44개 파일 동일      │
├──────────────────────────────────────────────────┤
│  Offset 72~103  : ASCII 식별자                     │
│  "PNE power supply schedule file"                 │
├──────────────────────────────────────────────────┤
│  Offset 336~    : 작성자 이름 (cp949 인코딩)       │
├──────────────────────────────────────────────────┤
│  Offset 656~1919: 메타데이터 영역 (용도 미확인)    │
├──────────────────────────────────────────────────┤
│  Offset 1920    : ▶ 스텝 블록 시작                 │
│  각 블록 652 바이트 고정                           │
│  total_steps = (file_size - 1920) / 652           │
└──────────────────────────────────────────────────┘
```

- **블록 크기 652B**: Half cell (12스텝), SEU4 RT (19스텝), Rss RT (37스텝) 등 모든 파일에서 동일
- **검증식**: `(file_size - 1920) % 652 == 0` → 44개 전체 파일 통과

### 검증 예시: SEU4 RT (2335mAh) Step 5

```
+12: 4550.0 mV  → 충전 상한 전압
+20:  466.0 mA  → 0.2C × 2335 = 467 mA ✓
+32:   48.0 mA  → ~0.02C (CV cutoff)
+88: 4600.0 mV  → 안전전압 = 4550 + 50 ✓
+96:  516.0 mA  → 안전전류상한 = 466 + 50 ✓
+100: 416.0 mA  → 안전전류하한 = 466 − 50 ✓
+104: 2569.0 mAh → 용량한도 = 2335 × 1.1 ✓
```

### CHG_CC의 +20 vs +32 관계

| 스케줄 | Step | +20 (current) | +32 (cv_cutoff) | 비율 |
|--------|------|---------------|-----------------|------|
| Half cell 3.45mAh | 8 | 0.34 | 0.07 | 0.2× |
| SEU4 2335mAh | 5 | 466 | 48 | 0.1× |
| SEU4 2335mAh | 14 | 2320 | 234 | 0.1× |

→ +32 = CV 전환 후 전류 하한값 (CC 모드에서도 설정)

### CHG_CC vs CHG_CCCV 구조 차이

| 필드 | CHG_CC (+offset) | CHG_CCCV (+offset) | 설명 |
|------|-------------------|---------------------|------|
| CC전류 | +20 | +20 | 동일 |
| 전압상한 | +12 | +12 | 동일 |
| CV cutoff 전류 | +32 | (없음) | CC만 CV cutoff 있음 |
| CV 전압 | (없음) | +28 | CCCV만 별도 CV전압 |

→ **CHG_CC**: "이 전압까지 이 전류로 충전, 전류가 +32값 이하면 종료"  
→ **CHG_CCCV**: "이 전류로 충전하다가 +12전압 도달 시, +28전압에서 CV 유지"

- **CHG_CC/CHG_CCCV**: 전압 = **+12**
- **DCHG_CC**: 전압 = **+16** (4바이트 뒤로 밀림!)

## 9. 안전 마진 패턴 (Safety Margin)

64~69% 스텝에서 안전한도값이 설정됨. 패턴:

| 필드 | 계산 규칙 |
|------|-----------|
| 안전전압상한 (+88) | 설정전압 + 50 mV |
| 안전전류상한 (+96) | 설정전류 + 50 mA |
| 안전전류하한 (+100) | 설정전류 − 50 mA |

→ 설정 전류/전압을 복원하며 역으로 검증 가능

## 10. 용량한도(+104)로 공칭용량 역산

```python
nameplate_mAh = capacity_limit / 1.1  # +104값 / 1.1
```

| 스케줄 | +104값 | 역산 | 실제 공칭 |
|--------|--------|------|-----------|
| Half cell | 11.0 | 10.0 | ~3.45 (풀셀 아님) |
| SEU4 RT | 2569.0 | 2335.5 | 2335 ✓ |
| GITT | 11.0 | 10.0 | 4.187 (half) |

→ 풀셀 수명 시험에서는 정확하게 동작. Half cell은 다른 용량 기준 사용.

## 11. 미확인/추가검증 필요 항목

1. **LOOP +56 vs +580**: 대규모 수명 시험 스케줄(수백 사이클)에서 실제 loop_count 확인 필요
2. **GOTO (0xFF06)**: 내부 구조 미분석 (점프 대상 오프셋 등)
3. **REST_SAFE (0xFF07)**: 필드 구조 미분석
4. **오프셋 656~1919 메타데이터**: 용도 미확인 (34, 4600.0 등 값 존재)
5. **DCHG_CC +28(safety_voltage)**: 일부 스케줄에서 전압 설정값보다 높은 값 → 과전압 안전 한도?

## 12. 결론 및 다음 단계

### 교차 검증 결과
- ✅ **652B 고정 블록**: 44개 전 파일 검증 통과
- ✅ **오프셋 1920 시작**: 전 파일 동일
- ✅ **타입 코드 7종**: 전 파일 일관, 충돌 없음
- ✅ **핵심 필드 위치**: 전압/전류/용량 — 타입별 100% 일관성
- ✅ **안전마진 패턴**: ±50 규칙 확인

### 파서 구현 준비도
- 충전/방전 스텝의 **전압, 전류, 용량** 추출 가능 → `_analyze_accel_pattern_pne()` 대체 가능
- CSV에서 역산하던 CC current를 .sch에서 직접 읽어 **정확도 향상**
- .sch 파싱 ~5ms vs CSV 분석 ~17ms → **3배 이상 속도 개선** 기대
