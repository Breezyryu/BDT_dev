### 1.2 Toyo 사이클 병합 병목 해결 ★

| 날짜 | 커밋 | 메시지 |
|------|------|--------|
| 02-06 | `c7b57fd` | toyo_cycle 병합 병목 해결 |
| 02-06 | `1179d70` | docs 추가 |
| 02-06 | `f073e04` | docs 추가 |
| 02-06 | `aac448b` | docs 추가 - 토요 사이클 처리 |

**문제**: `toyo_cycle_data()` 함수에서 연속된 동일 Condition 행을 병합하는 while 루프가 **O(n²)** 복잡도를 가짐. 4,664번의 `drop()` + `reset_index()` 호출로 전체 DataFrame을 매번 복사하여 2.36초 소요.

**원인 코드** (Before):
```python
while i < len(Cycleraw) - 1:
    if current_cond in (1, 2) and current_cond == next_cond:
        Cycleraw.loc[i + 1, "Cap[mAh]"] += Cycleraw.loc[i, "Cap[mAh]"]
        # ⚠️ 매번 전체 DataFrame 복사 → O(n²)
        Cycleraw = Cycleraw.drop(i, axis=0).reset_index(drop=True)
    else:
        i += 1
```

**해결** (After): `groupby()` 기반 벡터화 병합으로 단일 패스 처리.

| 구간 | Before | After | 개선도 |
|------|--------|-------|--------|
| Merge 루프 | 2.360s | 0.231s | **10.2배** |
| 전체 실행 | 2.374s | 0.245s | **9.7배** |

**학습 포인트**: Pandas에서 루프 내 `drop()`/`reset_index()`는 각각 전체 복사를 유발한다. 대신 병합 대상을 마스크로 식별 후 한 번에 처리하는 벡터화 방식을 사용해야 한다.

### 3.3 PNE 사이클 서치 최적화

| 날짜 | 커밋 | 메시지 |
|------|------|--------|
| 02-11 | `1356a95` | pne 사이클 서치 최적화 |

**설명**: PNE 바이너리 파일 검색 방식을 전체 스캔(O(n×m))에서 인덱스 캐싱(O(n+m))으로 전환.

```python
# Before: 각 사이클마다 전체 파일 목록 스캔 → O(n×m)
for cycle_num in cycle_list:
    for file in os.listdir(raw_path):
        if extract_cycle_num(file) == cycle_num:
            results.append(load_binary(file))

# After: 인덱스 1회 생성 후 O(1) 조회
cycle_index = {extract_cycle_num(f): f for f in os.listdir(raw_path)}
for cycle_num in cycle_list:
    if cycle_num in cycle_index:
        results.append(load_binary(cycle_index[cycle_num]))
```

**효과**: 1000 사이클 검색 시 약 **5배** 속도 향상.

**성능 최적화** (`3e8a18d`): `pd.concat()` 루프 내 반복 호출을 리스트 수집 후 1회 concat으로 변경. 100채널 처리 시 **25배** 속도 향상 (25초 → 1초).

```python
# Before: O(n²) — 매번 전체 복사
dfoutput = pd.DataFrame()
for df in data_list:
    dfoutput = pd.concat([dfoutput, df], axis=1)

# After: O(n) — 포인터만 수집 후 1회 concat
dfs_output = [df for df in data_list]
dfoutput = pd.concat(dfs_output, axis=1)
```
