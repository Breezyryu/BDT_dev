# 시간 복잡도
## 순차 탐색 시간 복잡도
- 최선: 1번
- 최악: n번
- 평균: (1+...+n)/n = n(n+1)/2/n = n/2+1/2
![[Pasted image 20251119095611.png]]

## 이진 탐색 시간 복잡도
- 최선: O(1)
- 최악: log_2(n)+1 >> O(log_2(n))
- 평균: O(log_2(n))

	 - 장점
	    - 안정한 정렬 방법

	    - 대부분위 레코드가 이미 정렬되어 있는 경우에 매우 효율적일 수 있다.
	- 단점
	    - 비교적 많은 레코드들의 이동을 포함한다.
	    - 레코드 수가 많고 레코드 크기가 클 경우에 적합하지 않다.
	- 최선의 경우
	    - 비교 횟수
	        - 이동 없이 1번의 비교만 이루어진다.
	        - 외부 루프: (n-1)번
	    - Best T(n) = **O(n)**
	- 최악의 경우(입력 자료가 역순일 경우)
	    - 비교 횟수
	        - 외부 루프 안의 각 반복마다 i번의 비교 수행
	        - 외부 루프: (n-1) + (n-2) + … + 2 + 1 = n(n-1)/2 = O(n^2)
	    - 교환 횟수
	        - 외부 루프의 각 단계마다 (i+2)번의 이동 발생
	        - n(n-1)/2 + 2(n-1) = (n^2+3n-4)/2 = **O(n^2)**
	    - Worst T(n) = **O(n^2)**
