# 개발

- DV : Design Verification (PIA ~ DVR)
- PV : Product Validation (DVR ~ PVR)
- PR : Production Readiness (PVR ~ PRA)

|     |                                    |              |                                                                                                                              |
| --- | ---------------------------------- | ------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| PKA | Project Kick-off Approval          | 프로젝트 발의 승인   | 라인업에 반영된 프로젝트에 대하여, 상품기획 外 프로젝트 CFT를 구성하고 DIA 승인 및 개발 진행을 위한 상품화 기획 단계의 시작을 공식화한다.                                           |
| DIA | Development Investment Approval    | 상품기획확정 승인    | 마켓조사, 기술조사, 고객 조사를 통해 마켓의 요구사항을 평가하고, 개발계획 단계 활동계획을 승인한다.                                                                    |
| PIA | Product Implementation Approval    | 개발구현승인       | 개발계획 수립 내용을 근거로 개발구현단계로의 진행을 승인한다.                                                                                           |
| DVR | Design Verification Review         | 설계자주평가       | 설계수행 결과가 사전에 기획된 개발 목표에 부합하도록 제품이 구현(기능, 동작, 성능 등)되었는지 개발  내부에서 평가하고 후속업무 수행 여부를 결정한다.                                       |
| PVR | Product Validation Review          | 개발검증검토       | PIA/DVR의 미결항목을 개선 적용하여 기능, 동작, 성능, 신뢰성 Test를 통해 설계 품질이 충분히 확보 되었는지를 검증하고 시생산/판매 준비가 완료되었는가를 점검하고 후속업무 수행 여부를 결정한다.           |
| PRA | Production Readiness Approval      | 양산승인         | 시 생산품(PR Set)의 결과를 통해 과제의 목표(ROI, 재료비, 일정, 공정품질 등) 달성, 양산 가능 여부를 승인한다.                                                       |
| SRA | Ship Release Approval              | 출하승인         | 판매를 목적으로 초도양산품의 결과를 통해 출하여부를 승인(SRA 승인은 초도양산품의 출하검사성적서로 대체 가능함)                                                              |
| PSA | Plusone Software Shipment Approval | 개선소프트웨어 출하승인 | SW Sign-Off이후 추가로 발행된 버전을 재다운로드하여 최신의 SW로 시장에 출시하기 위해 실시하고, 고객의 경험 개선을 위한 준비가 완료되었는가를 점검하여 승인한다.                             |
| MSA | Market Satisfaction Approval       | 시장만족 승인      | SRA 이후 2개월동안(Launching 기준 1개월) 양산 초기품질 안정화를 위해 약 2개월간 개발 주도로 다양한 채널로 입수되는 문제점을 해결하고 그 결과를 MSA 심의회에서 의결한 후 사후관리 업무를 CS로 이관한다. |
- FACA : Failure Analysis Corrective Action?
- HOQ : House of Quality
- DFMEA : Design Failure Mode and Effects Analysis
	- 불량 유형 – 불량 결과 – 관리기준 – 검출방법등을 명시
	- 심각도(severity), 발생빈도(Occurrence), 검출력(detection)등을 감안하여 R.P.N. 지수화하여 관리
	- Design FMEA(개발) 주관
- PFMEA : Process Failure Mode and Effects Analysis
	- Process FMEA(기술, 제조) 주관
- PMP : Project Management Plan
- IQC : Incoming Quality Control
- CTF : CRITICAL TO FACTOR/FUNCTION
