## 6. 강점

- **셀 해체 → 코인셀 재조립 → 양·음극 분리 평가** 라인을 박사과정 내에 직접 운영
  - → 6번 트랙 (측정 Campaign Owner) 의 "코인셀·삼전극 측정 + DB화" 즉시 contributor 가능
- **mechanistic 추론 (knee 원인 = 음극 계면 불균일도)**
  - → 4번 empirical fade 모델의 "외삽 (functional-form mediated extrapolation)" 측 박사급 peer challenge 통과 잠재력
- **희망 업무 3개가 부서 4·6·7번 트랙에 정확히 분포**
  - → 6번 (campaign) 으로 시작 → 4번 (수명 모델) 으로 확장 → 7번 (BMS·빅데이터) 진입 자연스러운 성장 경로 설계 가능
- **ML 도구 (PyTorch) 기본 경험**
  - → 7번 빅데이터 트랙 + AX 측 (2.1 AX 성과) contribution 가능

## 7. 약점·우려 사항

- **모델링 깊이**
  - "PyTorch 기본 모델 활용" 표현 → empirical fade 의 외삽 보장 (ADR-0003 functional-form mediated) 이나 EC modeling (P2D · identifiability · UQ) 경험은 미확인
  - 부서 5번 트랙 (전기화학 모델링) main contribution 은 어려울 수 있음 — 4·6·7번 위주 배치가 합리적
- **RUL 예측의 ML black-box 위험**
  - 박사급 peer challenge "측정 envelope 내 보장 + 외삽 시 functional form" 에 대한 인식 확인 필요
  - 후속 면접에서 **외삽 가능성 / 오차 정량화 / UQ** 인식 정도를 검증할 것
- **빅데이터 처리 인프라 경험**
  - 학교 환경 코인셀 데이터 vs 사내 BDT (4 제조사 × 8 PF 세대 × 다 form factor) 데이터 규모 차이 큼 — parquet · DB · pipeline 운영 경험 미확인
- **직무 비중**
  - 본인 본업 = 직무4 (수명·성능 해석) 메인. 후보자가 BMS 측 (직무3) 도 강한 관심 → **본업 우선 + BMS 는 7번 트랙 통해 단계적 진입** 으로 합의 필요

## 8. 후속 면접 (대면) 추천 질문

### 8.1 박사급 peer 검증

1. RUL 예측 모델의 **외삽 성능** — 학습 envelope 외 조건 (다른 화학·다른 form factor) 에서의 일반화는 어떻게 보장했는가?
2. **음극 계면 불균일도** 가 knee 의 원인이라는 추론을 어떤 정량 지표로 backing 했는가? (예: dV/dQ peak shift / DCIR 변화 / GITT D 변화)
3. **수명 예측 오차 정량화 (UQ)** — confidence interval 또는 Bayesian 처리 경험?

### 8.2 도구·도메인 깊이

4. PyTorch 외 **PyBaMM · COMSOL · 전기화학 시뮬레이터** 사용 경험 (5번 트랙 fit 가능성 검증)
5. **GITT · dV/dQ · ICA · DCIR** 분석 직접 수행 경험 (6번 트랙 SOP 적합성)
6. 학교 데이터 → 사내 데이터 (수십~수백 채널 × 수년 cycle) 로 확장 시 **데이터 파이프라인** 설계 경험

### 8.3 직무·성장 경로 정합

7. 직무4 (성능·수명 해석) 메인 + 직무3 (BMS) 보조 배치에 대한 동의 여부
8. 6번 트랙 (campaign owner — 일부 직접 + 업체 발주 + QA) 운영 시 **협력사 발주·QA cycle** 경험·태도

### 8.4 협업·OJT

9. 인계 받을 도구 (BDT 7·8 탭 등) 학습 곡선 — 대형 코드베이스 onboarding 경험
10. peer review · code review · 박사급 동료 challenge 수용 자세
