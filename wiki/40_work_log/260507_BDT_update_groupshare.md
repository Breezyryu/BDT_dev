## 한 줄 요약 (티저)

지난 한 달 동안 BDT가 바뀐 곳은 네 군데입니다.

1. **현황 탭** — 필터를 별도 sub-tab으로 분리, 분류기 v3 + STORAGE_CYCLE 카테고리 신설
2. **사이클 탭** — 정격전압(V_nom) 컬럼 추가 → 방전 Energy ratio (E / Q_nom·V_nom) 그래프 도입
3. **사이클·프로파일 결과 '데이터' 서브탭** — 채널×사이클 DataFrame을 엑셀처럼 보고 복사
4. **프로파일 분석 개편** — 사이클 바·데이터 범위·히스테리시스 6건 묶음 정리

마지막에 시간이 남으면 — Empirical 수명 예측을 "어떤 시험이 필요한가" 관점으로 간단히 짚겠습니다.

## 1. 현황 탭 — 필터링 상세

### 1.1 무엇이 달라졌나

- 기존 — 채널 리스트 sub-tab 안에 필터 버튼이 같이 있어서, 필터 상태와 채널 표시 상태가 섞여 보였습니다.
- 변경 — 현황 탭을 두 개의 sub-tab으로 쪼갰습니다. **채널 리스트** / **필터링**.
- 채널 리스트 쪽의 `btn_filter`는 제거. 필터 동작은 필터링 sub-tab에서만 동작합니다.

### 1.2 필터링 sub-tab의 기능

- 충방전기명 검색 — 장비 이름으로 바로 좁히기
- 분류기 v3 + STORAGE_CYCLE 카테고리 신설 — Phase 0-5 분류 결과 + 신규 STORAGE_CYCLE 시험을 별도 카테고리로 분리
- 빈 필터 진단 메시지 — 필터를 걸었을 때 결과가 없으면 어디서 비었는지 한 줄로 안내
- scope 미스매치 폴백 — 사용자가 잘못된 범위를 잡았을 때 인접 범위로 자동 전환

### 1.3 청중이 알아둘 포인트

- 필터·채널을 분리한 이유 — 두 동작이 같은 화면에 섞이면, 필터를 적용한 채로 채널을 바꿨을 때 사용자가 "왜 채널이 줄었지?"로 헷갈려서.
- STORAGE_CYCLE을 별도 카테고리로 뺀 이유 — 보관(저장) 사이클은 일반 cycling과 dQ/dV 형상이 다르므로, 분류 단계에서 섞이면 후속 분석(히스테리시스, fade fit)이 흐려집니다.

## 2. 사이클 탭 — 정격전압 + Energy ratio

### 2.1 정격전압(V_nom) 컬럼

- 경로 테이블에 **정격V** 컬럼 추가 — 셀별로 직접 입력
- 기본값 — 3.7 V → **3.9 V로 상향 조정** (최근 high-Ni NCM 셀에 맞춤)
- ylabel 의 `V_nom` 표기는 떼어내고 단순히 ratio로만 표시

### 2.2 방전 Energy ratio 그래프

- 정의 — `E_dchg / (Q_nom × V_nom)`
- 위치 — 사이클 상세 탭 (탭 2-4)
- 기존 그래프 (Discharge Capacity ratio, Charge Capacity ratio) 와 같은 ylim·ytick 으로 통일 → 한눈에 비교

### 2.3 왜 ratio인가

- 절대값 그래프는 셀 용량(Q_nom)에 따라 y축이 들쭉날쭉. 동일 plot에서 여러 셀 비교가 어려웠음.
- ratio 그래프는 1.0 근방에서 출발해 fade가 진행 → 셀 간·시험 간 직접 비교 가능
- 동일 ylim/ytick으로 맞췄으니 Dchg / Chg / Eng 세 그래프를 위아래로 놓으면 fade 양상의 정량 비교가 그대로 됩니다.

### 2.4 부수 정리

- 사이클 바 상세정보 패널 제거 — 화면 잠식 + 정보 중복이 있어서 UI 단순화
- 사이클 분류 정규화 — schedule 기반 분류와 휴리스틱 분류가 서로 다른 명칭을 쓰던 것을 통일

## 3. 사이클·프로파일 결과 — '데이터' 서브탭 신설

### 3.1 동기

- 그래프만 보면 "이 셀의 사이클 12 방전 용량이 정확히 얼마였지?"를 묻기 어려웠습니다.
- 분석 결과를 엑셀로 내보내지 않고도 즉석에서 숫자로 확인할 수 있어야 합니다.

### 3.2 사이클 분석 결과 → 데이터 서브탭

- 엑셀 시트 형태의 view (채널 × 사이클)
- 다중 셀 선택 + Ctrl+C로 복사 → 그대로 엑셀에 붙여넣기 가능

### 3.3 프로파일 분석 결과 → 데이터 서브탭

- (채널 × 사이클) DataFrame view
- **성능 — QTableWidget 에서 QTableView로 교체. 40초 → 20 ms** (약 2,000배)
- 대용량 시험에서도 즉시 로딩

### 3.4 시연 포인트

- 그래프 → 데이터 서브탭으로 전환 → 특정 채널·사이클 셀 다중 선택 → Ctrl+C → 엑셀 붙여넣기
- 한 사이클 내내 보던 그래프 값이 곧바로 숫자로 보이는 시연

## 4. 프로파일 데이터 분석 — 개편

> 이번 공유에서 손이 가장 많이 간 섹션입니다. 4월 28일 ~ 5월 5일 사이에 30+ commit이 들어갔고, 아래 네 갈래로 정리했습니다.

### 4.1 사이클 바 (경로 테이블) 개편

- TC 컬럼이 항상 편집 가능 — 토글 없이 바로 입력
- 경로 테이블 7열 → **6열**로 축소 (불필요 컬럼 제거)
- Excel 스타일 드래그 채우기 — 한 셀 입력 후 드래그로 아래로 채우기
- 다중 셀 복사·붙여넣기 — Ctrl+C / Ctrl+V 정상화
- 경로 교체 시 stale TC 자동 갱신 — 이전 경로의 TC 메타가 캐시에 남아있던 버그 해결

### 4.2 데이터 범위 분류 — Layer A 단일화

- 기존 — `_unified_pne_load_raw` 단계에 `data_scope` 분기가 있어서, 같은 raw 데이터를 scope별로 다시 읽는 비효율 + 분기 로직 누수
- 변경 — Layer A는 단일 raw 로딩만 담당, scope 분기는 view 단계로 이동
- 사용자 체감 — 같은 시험을 여러 scope (충전 / 방전 / CV)로 보더라도 한 번만 로딩
- '이어서' overlap 옵션은 충전 / 방전 데이터 범위에서 제거 → overlap 옵션 한 곳으로 통일

### 4.3 히스테리시스 분석 — 6건 통합 수정

> 가장 손이 많이 간 항목입니다. 한 commit으로 6 가지 회귀를 묶어서 정리.

| # | 항목 | 무엇을 고쳤나 |
|---|---|---|
| 1 | 페어링 | TC N + TC N+1 보완 phase 결합 (체크박스 + 진단 로그) |
| 2 | anchor | SOC 오프셋 anchor shift 보정 — 기준점이 어긋나던 케이스 |
| 3 | scaling | y축 스케일 일관화 — major / minor 그룹별 dim 처리 (Option B) |
| 4 | CV | CV 영역 필터링 robustness — saveok ON일 때만 long-format 빌드 |
| 5 | 분류 | RSS 측정 사이클이 HYSTERESIS 로 오분류되던 문제 |
| 6 | axis_mode | DOD 축 좌표 — 사이클+분리/연결 물리 좌표계 회복 |

추가로

- 히스테리시스 사이클 라벨/색상 — 깊이 기반 (Dchg/Chg X%) — major/minor 가 시각적으로 구분
- 히스테리시스 프리셋 — 단일 flow + Origin 호환 dQdV 시트 + Hysteresis_Analysis long-format 시트
- 부분 TC 선택 지원 (sweep PULSE_DCIR 케이스)

### 4.4 프로파일 옵션 체계 정리

- continuity + loop → **overlap 단일 옵션**으로 통합
- 사이클+분리+시간 조합은 비활성화 (결과값이 물리적으로 의미 없음)
- 옵션 4종 모델 spec 명시 + 다중경로 색상 G6 통일

### 4.5 시연 포인트

- 사이클 바에서 드래그 채우기 — 키 입력 줄이기
- 프로파일 분석 옵션 토글 → 히스테리시스 페어링 → axis_mode = DOD
- "이전엔 페어링이 깨졌는데 지금은 어떻게 보이는가" 비교 캡처

## 5. (시간 허락 시) Empirical 수명 예측 — 어떤 시험이 필요한가

> 모델 식이 아니라 **시험 매트릭스 관점**에서 짧게.

### 5.1 핵심 메시지

- Empirical fade 모델은 결국 **interpolation** 입니다. 측정 격자가 없으면 예측도 없습니다.
- 사용자가 알고 싶은 건 보통 "Q8 셀의 X 사이클·Y°C·Z DOD 조건에서 EOL이 언제인가" — 이 조건이 측정 매트릭스 안에 들어와 있어야 합니다.

### 5.2 필요한 시험 격자 (개념)

| 축 | 의미 | 일반적 격자 |
|---|---|---|
| 온도 | Calendar / Cycle 모두 | -10 / 0 / 25 / 45 / 60 °C |
| SOC (Calendar) | 보관 SOC | 30 / 50 / 80 / 100 % |
| DOD (Cycle) | 사이클 깊이 | 80% (full) / 50% / 30% |
| C-rate (Cycle) | 충·방전 속도 | 0.5C / 1C / 2C |

→ 이 격자의 외곽 (corner case) 만 측정해도 fade 모델은 fit 가능. 단, **외곽이 비어 있으면 그 영역은 외삽**이 됩니다.

### 5.3 BDT가 처리하는 부분

- 측정된 사이클 fade curve를 **자동으로 normalize → ratio 그래프** (위 2장 참고)
- Calendar / Cycle 시험을 **분류기 v3** 단계에서 분리 → 각각 다른 fit 흐름
- STORAGE_CYCLE 카테고리 분리 — 보관 + 사이클 혼합 시험을 따로 처리
- (후속) interpolation 결과를 시각화 — 격자 내부 / 외삽 영역을 색상으로 구분

### 5.4 그룹에 대한 부탁

- 신규 셀 평가 계획에서 **격자가 비지 않도록** 매트릭스 점검 부탁드립니다.
- 외곽 corner 만이라도 채워주시면, BDT 가 fade 모델 fit 까지 자동화합니다.
