# 갱신 가이드

- **항목 단위 추가** — 큰 업무 발생 시 해당 분기 + 해당 범주 안에 한 줄.
- **완료 표시** — 항목 우측에 `→` 결과 한 줄 또는 `✓` 표시.
- **분기 transition** — 다음 분기 시작 시 carry-over 항목 명시 (이전 분기에 → carry-over).
- **큰 업무만** — 일상 커밋·소소한 버그 수정·UI 미세조정은 제외.
- **분류 모호 시** — 우선 3번(모델링+데이터툴)에 흡수. 새 범주 필요하면 5번 신설 후 사용자 확인.
- **본 문서 위치 고정** — `wiki/40_work_log/26_수시피드백.md`. 이동·rename 시 메모리 reference 동시 갱신.

# Weekly Worklog 연동

- 일 단위 worklog 폐기 (W19 부터). 주 단위 단일 — `wiki/40_work_log/26_W{NN}.md` (ISO week).
- 주 종료(금) 시점 박제. 주 종합 섹션의 "4범주 측 매핑"이 본 분기 트래킹 문서로 흐른다.
- weekly worklog 의 큰 업무만 본 문서 분기 섹션에 흡수. 일상 commit·UI 미세조정은 weekly 에만 남기고 본 문서엔 미반영.
- 주 단위 → 분기 단위 transition flow:
  - W{NN}.md (일자별 + 주 종합) → 26_수시피드백.md (분기별 + 4범주 박제)
- 관련 메모리: [reference_weekly_worklog](../../../.claude/projects/C--Users-Ryu-battery-python-BDT-dev/memory/reference_weekly_worklog.md)
