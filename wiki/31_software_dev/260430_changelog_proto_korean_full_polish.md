## 1. 배경

직전 커밋(`7232bd4`)에서 PNE outer goto 영역만 윤문한 뒤, 이번에는 파일 전체를 대상으로 동일한 원칙을 적용했다.

검토 결과 이 파일은 점진적으로 다듬어진 코드라서 한국어 품질이 이미 매우 높았다.

- "을 위함", "에 의해", "로 인해", "함으로써", "되어진/되어져", "에 있어서", "에 대해서/대하여서" 등의 직역체 패턴 거의 0건
- "위해서는", "에 따라서", "것이다.", "의 경우에" 같은 어색한 구문도 거의 없음
- 명사형 종결("~함.", "~됨.", "~음)") 도 docstring 의 표준 보고체로 자연스러움

이런 환경에서는 보수적 접근 (의미 보존이 자연스러움보다 우선) 이 정답이라고 판단해, **명백히 비표준이거나 어색한 표현 약 20건만 정정**했다.

## 4. 의도적으로 윤문하지 않은 카테고리 (스킵 케이스)

### 4-1. 외부 시스템 연동 UI 라벨
- "스케쥴명" (`L17632 self.tb_info.setItemText`) — 비표준 표기지만 외부 호환 가능성으로 보존
- 카테고리 라벨 사전 (`'충전': '충전'`, `'사이클(ACCEL)': '사이클 (가속수명)'` 등) — 외부 데이터/시리얼라이즈 키
- "Goto 스텝", "Goto 반복횟수" — CTSEditor UI 인용

### 4-2. 도메인 약어와 짧은 라벨
- 1-3 단어짜리 짧은 트레일링 주석 (`# 충전`, `# 휴지`, `# 폴더 확인` 등) — 이미 명확
- DCIR/SOC/RPT/GITT 등 도메인 약어 — 손대면 안 됨
- "data 처리 class", "data 기본 처리" — 영어/한글 혼재 라벨이지만 이미 명확

### 4-3. 자연스러운 사용자 메시지
- "파일을 찾을 수 없습니다", "파일명에서 용량을 추출할 수 없습니다" — 사용자 표시 정중 어조 적절
- "오류가 발생하여 건너뛰었습니다" — 적절
- "정상 데이터는 정상적으로 표시됩니다" — 적절

### 4-4. 명사형 종결 docstring
- `"""전체 채널 메타데이터 캐시 초기화."""` — 보고체 명사형 자연스러움
- `"""...반환. 없으면 None."""` — 동일

### 4-5. 코드 식별자/format placeholder/docstring 의 인용 식별자
- 모든 변수명, 함수명, 클래스명, dict key, format placeholder ({var}, %s) 미변경
- backtick 안 식별자 (`goto_target_step`, `cycle_map`) 그대로 유지
