## 1. 핵심 결론 (TL;DR)

### 결론 1 — 데이터/스케줄은 정상. 분류기가 문제
240919 케이스의 TC 7-96 은 `parse_all_sch.py` 의 리치 분류기로 돌리면 `INIT → FORMATION(3) → RPT → CHARGE_SET → SOC_DCIR × 4 + ACCEL × 2` 로 **완벽히 분류**됨. UI 의 "분류 불가"는 **proto_.py 내장 분류기(6 카테고리)가 SOC_DCIR/ACCEL 조합을 인식 못함**이 원인.

### 결론 2 — 두 분류기 불일치
- `proto_.py` 내장(`_classify_single_pne_cycle`): **6 카테고리** (RPT, Rss, 가속수명, GITT, initial, unknown)
- `parse_all_sch.py`: **14 카테고리** (INIT, FORMATION, RPT, ACCEL, GITT_PULSE, SOC_DCIR, RSS_DCIR, RATE_TEST, KVALUE, CHARGE_SET, TERMINATION, HYSTERESIS_CHG, HYSTERESIS_DCHG, UNKNOWN)

두 분류기가 별도로 진화해와서 결과 불일치. parse_all_sch.py 가 UI 파이프라인에 연결되지 않음.

### 결론 3 — 리치 분류기도 UNKNOWN 9.4 %
parse_all_sch.py 기준으로도 1,745 loop group 중 **164건(9.4%) UNKNOWN**. 실험 폴더로 보면 **94/156 (60%)** 이 하나 이상 UNKNOWN 포함. 주된 미분류 군집:
- 장시간 심방전 (ECT 후반부 저장 준비용)
- N=3~19 짧은 수명 LOOP (현재는 N≥20 만 ACCEL 인식)
- DCHG_CCCV (SOC 세팅 방전)
- 30 m~1.5 h 단독 REST
- 복합floating 의 position=1 초기 방전

### 결론 4 — 용어 재정비 필요 (도메인 정합)
기존 `RSS_DCIR` 은 실은 **HPPC(Hybrid Pulse Power Characterization)** 에 해당. RSS 는 "확산저항까지 포함한 정상상태 저항"이라는 별개 물리량이며, 측정에는 긴 REST(≥30 m)가 필요. 두 개념 혼용되어 옴.

### 3.3 판별 결정 트리 (우선순위 순)

```
1. EC(End Condition) 플래그 기반
   ├─ EC=SOC_CHG(0x4800)  → 히스테리시스(충전)
   ├─ EC=DOD(0x0800)      → 히스테리시스(방전)
   └─ EC 다수(≥4) + N≥5   → SOC별 사이클

2. GITT (N≥10, n_steps≤3, 첫 REST≥1h, 전류≤C/10) → GITT

3. DCIR (N≥2, EC, dchg≥4, 짧은 REST) → DCIR
   ※ REST≥30m 이면 RSS 변종 — 데이터 확인 후 GITT(simplified) 로 분류 가능

4. 사이클 반복 (chg_count≥1 AND dchg_count≥1 AND N≥2)
   ├─ N=2~10 AND position≤2 → 사이클(FORMATION)
   └─ N≥2 (그 외)          → 사이클(ACCEL)

5. 단순 1회 사이클 (N=1, chg=1, dchg=1, n_steps≤8) → RPT

6. 단독 충전 (N=1, chg만)
   └─ → 충전(세팅)

7. 단독 방전 (N=1, dchg만)
   ├─ 긴 DCHG (t≥10800s=3h) + REST → 방전(저장준비)
   ├─ position==last                → 방전(종료)
   ├─ position≤1                    → 방전(초기)
   └─ (그 외)                       → 방전(세팅)

8. 단독 휴지·저장
   ├─ CHG_CP 포함                   → 저장
   ├─ REST≥7200s (2h) 단독          → 저장
   └─ REST 1800~7200s 단독          → 저장

9. FORMATION 폴백 (N=2~10, chg≥1, dchg≥1, position≤2) → 사이클(FORMATION)

10. UNKNOWN (위 전부 실패)
```
