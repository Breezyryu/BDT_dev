# BatteryDataTool - 패턴수정 탭 SOP
### PNE 충방전 패턴(Schedule) 수정 및 Toyo 변환

---

## 📌 1. 탭 개요

| 항목 | 내용 |
|------|------|
| **탭 이름** | 패턴수정 |
| **목적** | PNE CTSPro의 패턴 DB(.mdb)를 직접 수정하거나, PNE 패턴을 Toyo 형식으로 변환 |
| **대상 DB** | `Cycler_Schedule_2000.mdb` (Microsoft Access) |
| **핵심 테이블** | `Step` (스텝 상세), `TestName` (패턴명), `BatteryModel` (모델/폴더) |

## 📌 3. 조작 순서 (SOP)

### Step 1: 패턴 DB 경로 설정

| 항목 | 설명 |
|------|------|
| **기본 경로** | `C:\Program Files\PNE CTSPro\DataBase\Cycler_Schedule_2000.mdb` |
| **위젯** | `ptn_ori_path` (직접 편집 가능) |
| **연결 방식** | pyodbc + Microsoft Access Driver |

> 경로가 없으면 각 수정 버튼 클릭 시 파일 선택 대화상자가 자동으로 열림

---

### Step 2: 패턴 리스트 불러오기

1. **[패턴 리스트 Load]** 버튼 클릭
2. MDB 접속 → `TestName` + `BatteryModel` 테이블 JOIN
3. 패턴 목록이 `ptn_list` 테이블에 표시됨

| 컬럼 | DB 소스 | 설명 |
|------|---------|------|
| 패턴폴더 | `BatteryModel.ModelName` | 모델/폴더 분류명 |
| 패턴이름 | `TestName.TestName` | 스케줄(패턴) 이름 |
| 비고 | `TestName.Description` | 설명/메모 |
| TestID | `TestName.TestID` | 내부 식별자 (숨김) |

> 정렬: `No` → `TestNo` 순 오름차순

---

### Step 3: 패턴 선택

- 리스트에서 수정할 패턴 **행(Row)을 클릭** (복수 선택 가능)
- 선택 시 `ptn_df_select`에 해당 TestID 목록 저장
- **반드시 패턴을 선택한 후** 오른쪽 수정 버튼 사용

---

### Step 4: 원하는 수정 실행

오른쪽 패널의 수정 그룹 중 원하는 항목 선택 → 값 입력 → 버튼 클릭

## 📌 4. 수정 기능 상세

### 4-1. 전류 일괄 변경

| 항목 | 위젯 | 기본값 | 설명 |
|------|------|--------|------|
| 방전 최대 C-rate | `ptn_crate` | 1.0 | 기준 방전 C-rate |
| 변경할 용량(mAh) | `ptn_capacity` | 4000 | 새 기준 용량 |

**동작 원리:**
```
base_capacity = MAX(Iref, StepType=2) / ptn_crate
→ 전체 Step의 Iref = Iref / base_capacity × ptn_capacity
→ 전체 Step의 EndI = EndI / base_capacity × ptn_capacity
```

| Coin cell 모드 | 전류 변환 | 비고 |
|----------------|----------|------|
| OFF | `int()` 정수 절삭 | 일반 셀 (mA 단위) |
| ON | `round(..., 3)` 소수점 3자리 | 코인셀 (μA~mA 단위) |

---

### 4-2. 개별 파라미터 변경 (Find & Replace 방식)

| 기능 | DB 컬럼 | 기존값 위젯 | 변경값 위젯 | 단위 |
|------|---------|------------|------------|------|
| **충전 CV 전압 변경** | `Vref_Charge` | `ptn_chgv_pre` (4470) | `ptn_chgv_after` (4500) | mV |
| **충방전 전류 변경** | `Iref` | `ptn_refi_pre` (2000) | `ptn_refi_after` (1000) | mA |
| **방전 CV 전압 변경** | `Vref_DisCharge` | `ptn_dchgv_pre` (3000) | `ptn_dchgv_after` (2750) | mV |
| **Cut-off 전류 변경** | `EndI` | `ptn_endi_pre` (100) | `ptn_endi_after` (50) | mA |
| **Cut-off 전압 변경** | `EndV` | `ptn_endv_pre` (3000) | `ptn_endv_after` (3300) | mV |

**동작 원리 (동일 패턴):**
```
SELECT된 TestID에 대해:
  1) 강제 정수화: UPDATE SET col = int(col / 1 * 1)  ← 문자 인식 오류 방지
  2) 값 치환:     UPDATE SET col = 변경값 WHERE col = 기존값
```

> Coin cell 모드: **전류** 관련 (`Iref`, `EndI`)만 `round(...,3)` 적용 / **전압** 관련은 동일

---

### 4-3. Step 변경 (전체 밀기)

| 항목 | 위젯 | 기본값 | 설명 |
|------|------|--------|------|
| 변경할 스텝 | `ptn_step_pre` | 10 | 기존 스텝 번호 |
| 추가로 더할 스텝 | `ptn_step_after` | 5 | 오프셋 추가값 |

**동작 원리:**
```
Step 테이블의 Value2 컬럼에서:
  " 10 " → " 15 " (공백 포함 패턴 치환)
  → Loop 대상 스텝 번호 등이 일괄 이동됨
```

---

### 4-4. Toyo 패턴 변환

| 항목 | 설명 |
|------|------|
| **버튼** | `ptn_toyo_convert` ("Toyo 패턴 변환") |
| **기능** | PNE MDB의 Step 데이터를 **Toyo PATRN 파일 형식**으로 변환 |

**조작 순서:**
1. 패턴 리스트에서 변환할 패턴 선택
2. [Toyo 패턴 변환] 클릭
3. 출력 폴더 선택 대화상자
4. 패턴 번호 입력 (PATRN{N}.1)
5. 선택 패턴마다 자동 번호 증가하며 파일 생성

**생성 파일:**

| 파일명 | 설명 |
|--------|------|
| `PATRN{N}.1` | 패턴 본체 (헤더 + 서브스텝 라인들) |
| `Patrn{N}.option` | BaseCellCapacity 정보 |
| `Patrn{N}.option2` | 라인별 active/rest 플래그 |
| `Fld_Puls.DIR` | 펄스 디렉토리 파일 |

**PNE → Toyo StepType 변환 매핑:**

| PNE StepType | 설명 | Toyo 변환 |
|-------------|------|-----------|
| 1 | 충전 | LEFT 서브스텝 (CC-CV 또는 CC) |
| 2 | 방전 | RIGHT 서브스텝 (방전) |
| 3, 4 | 휴지/OCV | REST 서브스텝 |
| 5 | Impedance | 짧은 방전 펄스 |
| 8 | Loop | 이전 라인에 loop 정보 부착 |
| 9 | Continuation | 이전 맥락에 따라 충전/방전 |
| 6 | End | 건너뜀 |

## 📌 5. 데이터 흐름

```
[패턴 리스트 Load] 클릭
    │
    ▼
MDB 접속 (pyodbc + Access Driver)
    ├─ TestName 테이블
    └─ BatteryModel 테이블 → JOIN on ModelID
    │
    ▼
ptn_list 테이블에 패턴 목록 표시
    │
    ▼
사용자: 패턴 행 선택 (TestID 수집)
    │
    ├─── [수정 버튼] ──► MDB Step 테이블 직접 UPDATE (SQL)
    │                     → Iref, EndI, Vref_Charge, Vref_DisCharge, EndV, Value2
    │                     → conn.commit() → DB에 즉시 반영
    │
    └─── [Toyo 변환] ──► Step 데이터 SELECT
                          → _pne_steps_to_toyo_substeps() 변환
                          → PATRN{N}.1 / option / option2 / Fld_Puls.DIR 생성
```

## 📌 6. 주의사항

| 항목 | 설명 |
|------|------|
| **DB 직접 수정** | 수정 버튼은 MDB를 **즉시 commit** → 되돌리기 불가, 사전 백업 필수 |
| **패턴 선택 필수** | 패턴 미선택 시 수정 버튼은 동작하지 않음 (silent skip) |
| **Access Driver** | 32bit/64bit 드라이버 일치 필요 (`Microsoft Access Driver (*.mdb, *.accdb)`) |
| **Coin cell 체크** | 전류 계산 정밀도가 달라짐 → 코인셀이면 반드시 체크 |
| **복수 패턴 선택** | 여러 패턴 선택 시 모든 선택된 TestID에 동일 수정 적용 |
| **Toyo 변환** | Loop 스텝은 PNE 스텝 번호 → Toyo 라인 번호 매핑 자동 계산 |
| **단위** | 전압은 **mV**, 전류는 **mA** 단위로 입력 |
