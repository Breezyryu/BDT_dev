## 1. 배경

### 1.1 현재 BDT 의 GITT 처리 상태
- **입력**: `GITT(full)` 카테고리로 분류된 60 loop group (Phase 1 분류기 기준)
- **처리**: voltage·current profile 시각화 + dQ/dV 플롯만 제공
- **결과물**: 정량적 D_s 값 없음 — 연구자가 수기로 Excel 계산

### 1.2 F1 차용 시 기대 효과
| 지표 | 현재 | F1 이식 후 |
|------|------|-----------|
| D_s 추출 | 수기 (Excel, per-pulse) | 자동 (SOC 배열 × 수십 pulse) |
| 실험당 처리 시간 | 2-4시간 | < 1초 |
| 결과 표준화 | 연구자별 상이 | 단일 알고리즘 |

| 반셀 지원 | 프로파일만 | D_s 양·음극 개별 |

## 2. 물리·수학 기반

> 상세 유도·SOC 의존성·GITT 프로토콜 설계 원리는 **[[260419_GITT_확산계수_추출]]** 참조. 본 문서는 구현에 필요한 최소 요약만 수록.

### 2.1 핵심 공식
```
D_s = (4/9π) · (r · dE/dt / dU/d√t)²
```
- r: 입자 반경 [m]
- dE/dt: pulse 간 E_eq 변화율
- dU/d√t: pulse 내 √t vs V 선형회귀 기울기

### 2.2 측정 개념 도식
```
V(t)
 ▲
 │  ┌─pulse─┐                        ┌─pulse─┐
 │  │  ●●   │                        │  ●●   │
 │  │    ●●─┤  rest(relaxation)      │    ●●─┤
 │──┘      ●─────────●━━━━━━━━━━━────┘      ●─────●━━━...
 │                   ↑ E_eq,1                     ↑ E_eq,2
 │                   └─── dE = E_eq,2 − E_eq,1 ───┘
 │
 └──────────────────────────────────────────────► t

pulse 내부 확대 (√t 축):
 V ▲
   │   Sand slope m
   │  ╲___                    m = dU/d√t
   │     ╲___                 b = E_eq (회귀 절편)
   │         ╲___
   │             ╲___━━━━━━━
   │                        
   └────────────────── √t
         tmin      tmax
```

### 2.3 알고리즘 핵심 단계
1. **State 분류**: Amps>0→C, Amps<0→D, Amps=0→R
2. **Pulse 감지**: `(State != R) & (prev State == R)` → 누적합
3. **SOC 계산**: `cumtrapz(|I|, t/3600) / max` (정규화)
4. **회귀**: 각 pulse 의 [tmin, tmax] 구간 `√StepTime` vs `Volts` 선형회귀 → `m, b`
5. **dE/dt 산출**: pulse 시작점 V 를 `cumsum(dt_pulse)` 로 gradient
6. **D_s 계산**: 위 공식 적용, SOC 정렬 반환

### 3.1 함수 시그니처
```python
def extract_params(

    tmin: float = 1,         # 회귀 하한 [s]

    return_all: bool = False,
) -> pd.DataFrame | tuple[pd.DataFrame, pd.DataFrame]:
```

### 3.2 라인별 로직 분석

#### 3.2.1 입력 검증
```python
_check_columns(data, {'Seconds', 'Amps', 'Volts'})
_check_only_one(
    [any(Amps>0), any(Amps<0)],
    "data cannot include both charge and discharge segments.",
)
```
→ 충/방전 혼재 데이터는 **에러**. BDT 래퍼에서 **방향별 자동 분할** 필요.

#### 3.2.2 State 분류 + SOC
```python
df['State'] = 'R'
df.loc[Amps>0, 'State'] = 'C'
df.loc[Amps<0, 'State'] = 'D'

Ah = cumulative_trapezoid(|Amps|, Seconds/3600, initial=0)
df['SOC'] = Ah/Ah.max() if charging else 1-Ah/Ah.max()
```
- `Amps==0` 을 R 로 판별 — float 비교 주의: BDT 원본에서 rest 전류가 ±0.0001 mA 정도 남을 가능성. `abs(Amps)<threshold` 필요.
- SOC: 충전은 0→1, 방전은 1→0 방향.

#### 3.2.3 Pulse 감지
```python
pulse = (df.State != 'R') & (df.State.shift(fill_value='R') == 'R')
df['Pulse'] = pulse.cumsum()
```
- 0 = 초기 rest 구간
- 1, 2, ... = 각 pulse (rest→active 전이 카운트)

#### 3.2.4 Relative StepTime
```python
df['StepTime'] = groupby(['Pulse','State'])['Seconds'].transform(
    lambda x: x - x.iloc[0]
)
```
→ 각 pulse·state 시작 기준 상대시간. 회귀 입력.

#### 3.2.5 불완전 사이클 제거
```python
if df.iloc[-1]['State'] != 'R':
    df = df[df['Pulse'] != df['Pulse'].max()]
```
→ 마지막 pulse 가 rest 로 종료 안되면 제외.

#### 3.2.6 회귀 루프 (핵심)
```python
regression = None
for idx, g in df.groupby('Pulse'):
    if idx == 0: continue   # 초기 rest 스킵
    
    rest = g[g.State == 'R']
    pulse = g[g.State != 'R']
    
    pulse_window = pulse[(pulse.StepTime >= tmin) & (pulse.StepTime <= tmax)]
    x = sqrt(pulse_window.StepTime)
    y = pulse_window.Volts
    
    if len(x) <= 1:
        x, y = [0,1], [NaN, NaN]   # 회귀 불가능
    
    result = linregress(x, y)
    regression.append({
        'Pulse': idx,
        'Eeq': result.intercept,
        'Eeq_err': result.intercept_stderr,
        'dUdrt': result.slope,
        'dUdrt_err': result.stderr,
        'dt_rest': rest dt,
        'dt_pulse': pulse dt,
    })
```

#### 3.2.7 D_s 계산
```python
stats['dEdt'] = np.gradient(stats.Volts, np.cumsum(stats.dt_pulse))
params = DataFrame({
    'SOC': stats.SOC,
    'Ds':  4/9/π * (radius * stats.dEdt / stats.dUdrt)**2,
    'Eeq': stats.Eeq,
})
return params.sort_values('SOC')
```

### 3.3 주목할 구현 특징

**장점**:
- ✅ 결측치 robust 처리 (`len(x)<=1` → NaN)
- ✅ 선형회귀 표준편차까지 제공 (`Eeq_err, dUdrt_err`) — F8 uncertainty 연동 용이
- ✅ `return_all` 옵션으로 디버그용 상세 stats 제공

**한계**:
- ❌ tmin=1s 고정 — 샘플링 주기 50ms 인 BDT PNE 는 무의미한 수치
- ❌ state 분류 시 부동소수 허용오차 미고려
- ❌ iR drop 자동 제외 없음 — tmin=1s 로 어느 정도 커버되지만 명시적 보정 부재
- ❌ 반셀 GITT (I=0.4 mA 수준) 도 처리되지만 성능 검증 필요

### 4.2 BDT 어댑터 설계

#### 4.2.1 데이터 로더
```python
# _common.py (신규)
def pne_to_gitt_frame(channel_path: str,
                      tc_range: tuple[int, int] | None = None,
                      ) -> pd.DataFrame:
    """PNE SaveEndData → ampworks 입력 포맷.
    
    cycle_map 기반으로 GITT 구간만 잘라서 반환.
    단위: Seconds [s], Amps [A], Volts [V].
    """
    raw = _cached_pne_restore_files(channel_path)[0]
    # col 17=StepTime[0.01s], 9=Current[uA], 8=Voltage[uV], 27=TotlCycle
    if tc_range:
        raw = raw[raw[27].between(*tc_range)]
    
    # rest 임계값 — PNE 측정 노이즈 허용
    current_A = raw[9] / 1e6
    rest_mask = abs(current_A) < 1e-4   # 0.1 mA 이하 = rest
    current_A = current_A.where(~rest_mask, 0.0)
    
    return pd.DataFrame({
        'Seconds': (raw[19] / 100).values,      # TotTime
        'Amps':    current_A.values,
        'Volts':   (raw[8] / 1e6).values,
    })
```

#### 4.2.2 방향별 자동 분할
```python
def split_by_direction(df: pd.DataFrame) -> dict[str, pd.DataFrame]:
    """한 시험이 charge/discharge 혼재면 각각 분리."""
    has_chg = (df.Amps > 0).any()
    has_dchg = (df.Amps < 0).any()
    if has_chg and not has_dchg:
        return {'charge': df}
    if has_dchg and not has_chg:
        return {'discharge': df}
    # 혼재: 방향 전환점 기준 분할
    direction_change = (np.sign(df.Amps).diff().abs() > 1)
    splits = np.split(df, np.where(direction_change)[0])
    return {
        f'seg_{i}_{"C" if s.Amps.max()>0 else "D"}': s
        for i, s in enumerate(splits) if len(s) > 10
    }
```

#### 4.2.3 본 함수 래퍼
```python
# gitt_params.py
def extract_gitt_params_from_bdt(
    channel_path: str,
    radius_m: float,
    tc_range: tuple[int, int] | None = None,
    tmin: float = 1.0,
    tmax: float = 60.0,
) -> dict[str, pd.DataFrame]:
    """BDT 채널 경로 → SOC-Ds 테이블 (방향별)."""
    df = pne_to_gitt_frame(channel_path, tc_range)
    segments = split_by_direction(df)
    
    results = {}
    for direction, seg in segments.items():
        try:
            params = ampworks_extract_params(seg, radius_m, tmin, tmax)
            results[direction] = params
        except Exception as e:
            _logger.warning(f'{direction}: {e}')
            results[direction] = None
    return results
```

### 6.1 BDT 특수 리스크 — Rest 판별

BDT PNE `.cyc` 의 `StepType` 컬럼은 정확히 3=rest 를 제공하므로, **ampworks 의 `Amps==0` 판별보다 StepType 기반이 더 robust**. 래퍼에서 StepType 을 우선 사용하고 Amps 는 검증용으로만.

## 7. 확장 방향

### 7.1 Phase 후보 (F1 이식 후)
1. **반셀 metadata 파일** — `_common/electrode_presets.yaml`:
   ```yaml
   NMC622:     {r_p_um: 5.0, n: 1, LFP_flag: false}
   NMC811:     {r_p_um: 3.5, n: 1, LFP_flag: false}
   graphite:   {r_n_um: 12.0, n: 1, LFP_flag: false}
   LFP:        {r_p_um: 0.5, n: 1, LFP_flag: true}  # 경고 대상
   ```
2. **온도 보정**: Arrhenius 로 `D_s(T_ref)` 정규화
3. **GITT+ICI 통합 출력** — 두 기법을 동일 채널에서 돌려 교차검증

### 7.2 Advanced — 전체 pulse 해 기반 피팅
현재 semi-infinite 근사는 τ_p < r²/D_s 가정. 이를 벗어나면 무한 급수 해 (Doyle-Fuller-Newman) 로 전체 pulse 를 피팅해야 정확. 장기 과제.

### 7.3 PyBaMM 연동
추출한 D_s(SOC) 테이블을 **PyBaMM SPM/P2D 모델 파라미터로 바로 주입** 가능. BDT → 시뮬레이션 파이프라인 완성.

## 10. 최종 권고

✅ **F1 은 가장 낮은 리스크·가장 높은 가치의 첫 포팅 대상**.
- 물리·수식 안정 (1977 공식 48년째 산업 표준)
- BDT 실데이터 풍부 (≥ 1000 pulse)
- 구현 단순 (100줄 이내)
- 독립 검증 경로 명확 (synthetic + 문헌)

1. 본 분석 리뷰 완료 시 → `tools/ampworks_port/f1_gitt/` 생성
