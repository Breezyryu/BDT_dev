## 배경

240919 SOC별DCIR 시험에서 "사이클 7~96 (90cy) 분류 불가" 메시지 발생. 전수 조사 결과:
- `proto_.py` 내장 분류기는 6 카테고리(RPT / Rss / 가속수명 / GITT / initial / unknown) 만 지원
- `parse_all_sch.py` (별도 스크립트) 는 14 카테고리로 분류 가능하지만 UI 파이프라인에 연결 안됨
- 도메인 정합성 문제: 기존 `RSS_DCIR` 은 실제로 HPPC 측정이지 RSS 측정이 아님. RSS 는 확산저항 포함 정상상태 저항(측정에 긴 REST 필요)

사용자 결정:
1. parse_all_sch.py 로직 검증 후 proto_.py 이식 (조건부 승인)
2. 9종 한글 카테고리 네이밍 (충전/방전/저장/사이클/RPT/GITT/HPPC/DCIR/SOC별사이클) + 위치·목적 서브태그 유지
3. 히스테리시스 별도 카테고리 유지
4. Toyo 포함
5. HPPC 는 현 데이터셋에 없어 실질 9종 (히스테리시스 포함)

## 발견된 이슈 (Phase 1 이식 전 해결 필요)

### 이슈 1 — 가속수명 → SOC별 사이클 오분류
240919 케이스의 #7, #8 (ACCEL, TC 27-71) 이 신 분류기에서 "SOC별 사이클" 로 오분류됨.
**원인**: Rule 1 (`EC≥4 AND N≥5 → SOC별 사이클`) 조건이 광범위.
**해법**: `len(ec_values) ≥ 3` (서로 다른 SOC 레벨 3개 이상) 조건 추가.

### 이슈 2 — 남은 UNKNOWN 9 패턴 중 5건 구제 가능
1. 장시간 CHG_CC (t≥86400s) 단독 → 저장
2. 반셀 GITT 변형 (I=0mA + REST 1h × 120) → GITT(simplified)
3. 장시간 방전 분류 임계값 조정
4. `UNK_0x0006` TYPE_CODE 파서 보완
5. 짧은 REST 임계값 재검토

보강 후 **UNKNOWN 2.2 % → <1 %** 예상.
