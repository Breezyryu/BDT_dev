## 2. BDT — 4대 미션 통합 플랫폼

### 2.1 왜 BDT 가 중심에 오는가

- 과거: 각 엔지니어가 **개별 스크립트**(Matlab/Python/Excel) 로 수명·프로파일 분석 → 결과 포맷 불일치, 재현성 낮음
- 현재: **하나의 GUI 도구**에 ① raw 데이터 로딩 ② 분석 ③ 시뮬레이션 ④ 수명 예측까지 통합
- 효과:
  - 동일 입력 → 동일 분석 경로 → **재현성 확보**
  - 10개 모델 비교 시 로딩/분석 시간 **20~50초 → 수 초** (DB 연동 시)
  - 신입 엔지니어 **학습 곡선 단축** (스크립트 찾아다닐 필요 없음)

### 2.2 현재 BDT 탭 구조

```
┌─ 현황/필터링 ─┬─ 사이클데이터 ─┬─ 파형수정 ─┬─ 셋 결과 ─┬─ dVdQ 분석 ─┐
│              │               │            │          │             │
└─ Eu 수명예측 ─┴─ 승인 수명예측 ─┴─ 실사용 수명예측 ─┴─ 전기화학 시뮬레이션 ─┘
```

| 탭 | 4대 미션 매핑 | 상태 |
|----|---------------|------|
| 현황/필터링 | (공통 기반) | **v4 안정화 완료 (4/20)** |
| 사이클데이터 | 공통 기반 → dV/dQ·수명 입력 | 파이프라인 리팩터 완료 (4/12) |

## 3. 미션별 BDT 현황 상세

### 3.1 미션 1 — 소재 전기화학 물성 DB화

**목표**: 코인/3전극 측정 결과를 **DB에 적재** → SELECT 한 번으로 다중 모델 비교

**BDT 역할**:
- `.github/instructions/database.instructions.md` 기반 **PostgreSQL 스키마** 설계 (product / test_group / cycle_summary)
- BDT 클라이언트에 `db_connector` 모듈 (읽기 전용)
- DB 서버는 별도 NAS/서버에 배치 → 다수 사용자 동시 접근

**1분기 실적** (2026-03-20 plan 수립 후):
- ✅ 물성 파라미터 / DB 스키마 1차 설계
- ✅ 코인/3전극 측정 SOP 문서화 (`260322_SOP_convert_reliability_to_csv.md`)
- ✅ 신뢰성 `.xls` → CSV 변환 파이프라인 (Fasoo DRM 대응 완료)

**2분기 계획**:
- DB 데이터 관리 UI 를 BDT 안에 연동 (신규 탭)
- 업체 수급 물성값 → DB 1차 적재
- Q8 main/sub 스펙시트 입수 후 반영

**Risk**:
- DRM 환경에서 `.xlsx` 출력이 막힘 → 번들 txt + 차트 PNG 재설계로 회피 (4/15~20 개편 완료)

### 3.2 미션 2 — dV/dQ 양음극 분리 (열화 모드)

**목표**: 풀셀 dV/dQ 를 양·음극 성분으로 분리 → **LLI (리튬 재고 손실) / LAM (활물질 손실)** 정량화

**BDT 역할**: `dVdQ 분석` 탭
- 풀셀 곡선 스무딩 (저주파 노이즈 제거)
- 단극 OCV 모델 (Gr / Si 음극, NMC 양극) 매칭
- 열화 셀의 slip/mass 차이로 LAM / LLI 분리

**1분기 실적**:
- ✅ dVdQ 스무딩 로직 이식 (규섭님 주도)
- ✅ 단위 음극 (Gr / Si) OCV 모델 구축
- ✅ M2 양음극 코인셀 data (@ S.LSI) 1차 확보

**2분기 계획**:
- 풀셀 ↔ 단극 **피크 슬라이딩/매칭** 알고리즘 도입
- 핵심 열화 모드 (LLI, LAM) 자동 추출
- 3분기: Tool 내부 dVdQ 분석 탭 최종 통합, 분리 정확도 개선
- 4분기: 다수 사이클 열화 트렌드 자동 분석

**데이터 의존성**:
- 양음극 OCP profile 별도 확보 필수 (현재 M2 우선, M1/Q8 순차 확보)

### 3.3 미션 3 — 성능 시뮬레이션 (전기화학)

**목표**: 온도·충전 조건별 성능 예측 + 급속충전 리튬 석출 risk 사전 검출

**모델 선택**:
- **PyBaMM** (BSD-3, OSS 표준) — BDT 내부 표준 엔진으로 확정
- Autolion — 레퍼런스용 유지
- 내장 모델: **P2D / SPMe / SPM** 3단계 (복잡도 ↔ 정확도 trade-off)

**BDT 역할**: `전기화학 시뮬레이션` 탭
- 프리셋: `Chen2020` 등 공개 파라미터셋 즉시 로드
- 운영: CC/CCCV 충방전, **GITT / HPPC**, 커스텀 스텝
- 출력: 양·음극 OCP, 표면 Li 농도, 음극 potential (리튬 석출 margin = 0.02 V 확인용)

**1분기 실적**:
- ✅ 오픈소스 기반 모델 UI 구축 (탭 단위 통합)
- ✅ 충/방전 · GITT · HPPC 시뮬레이션 구축 (문헌 기반 파라미터)
- ✅ **전기화학 OSS 14종 비교 딥서치 완료 (4/19)**:
  - P2D 9종 / PINN 3종 / MD 2종 비교
  - 결과: PyBaMM 유지 + **PyBOP 통합** (역추정/캘리브레이션), PETLION.jl 속도 참고, PINNSTRIPES 손실 구조 참고
  - 라이선스 위험군 식별: cideMOD / CIDETEC PINN = AGPL → 직접 포함 불가
- ✅ PyBaMM 변수 전수 조사 PPT 자료 (4/11)

**2분기 계획**:
- 기본 베이스 모델 **캘리브레이션** (실측 ↔ 시뮬레이션 일치)
- **Arrhenius 온도 의존성** 저/고온 부여
- 3분기: 급속 스텝 충전 가상 시나리오 구현 → 리튬 석출 risk 도출 SW 화

### 3.4 미션 4 — 수명 예측 (Empirical 모델)

**목표**: 평가 환경(온도·C-rate·하한 SOC·저장)별 수명 파라미터 도출 → EOL 예측 자동화

**평가환경 매트릭스**:

| 축 | 조건 |
|----|------|
| 온도 | 15 ~ 45 ℃ |
| 복합 | 15 ~ 45 ℃ + 충전 후 5 hr / 12 hr |
| 방전 C-rate | 0.2 C ~ 1.0 C |
| 하한 SOC | 0 ~ 55 % |
| 저장 | SOC 105 ~ 95, 23 ~ 45 ℃ |

**BDT 역할**: `승인 수명예측` / `실사용 수명예측` 탭
- 평가환경 matrix → **folderble** 구조로 파라미터 자동 도출
- 상품화(승인) 수명: PF / 업체별 수명 trend
- 사용자(실사용) 수명: 상위 유저(고온·만충) 기준 시나리오

**1분기 실적**:
- ✅ 온도별 열화 파라미터 도출 (folderble)
- ✅ Cell Capacity / SET Capacity / Swelling trend 1500일 예측 곡선 구현

**2분기 계획**:
- **Si 하한 전압 열화 계수 추가** (현재 Si 고함량 셀에서 편차 큼)
- 승인/실사용 시나리오 EOL 예측 기능화
- **온도별 수명 데이터 수급** (23, 35, 45 ℃)
- 3분기: 다조건 결과 비교 + 수명 예측 자동화 리포트 산출

2. **풀셀 ↔ 단극 매칭** (미션 2) — 피크 슬라이딩 알고리즘, LLI/LAM 자동 추출
3. **기본 베이스 모델 캘리브레이션** (미션 3) — 실측 vs Sim 편차 목표 ≤ 3 %
4. **Si 하한 전압 열화 계수** (미션 4) — Si 고함량 셀 예측 오차 축소

### 5.2 병행 과제

- Arrhenius 온도 의존성 추가 (미션 3)
- 승인/실사용 EOL 예측 시나리오 (미션 4)
- 온도별 수명 데이터 수급 (23/35/45 ℃)
- PyBOP 통합 기초 작업 (미션 3, 4 연계)

### 5.3 관리 항목

- **라이선스 경계**: AGPL OSS 는 참고만, 코드 직접 포함 금지
- **DRM 호환 포맷**: 신규 산출물은 번들 txt 기준으로 공유
- **테스트 커버리지**: 신규 분석 기능은 E2E 테스트 추가를 기본 룰로
