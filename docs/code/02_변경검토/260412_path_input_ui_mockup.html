경로 입력 UI — UX 노트

1. 로드/저장/삭제 버튼의 발견성

현행은 테이블 우측에 24×24 아이콘 3개가 수직 나열. 아이콘만 있고 텍스트 라벨이 없어 처음 쓰는 사용자는 기능을 추측해야 합니다. 특히 삭제(🗑)와 저장(💾)이 인접하여 오조작 위험이 있습니다.

개선: 버튼을 체크박스 행 우측으로 이동하고, 아이콘+텍스트 조합으로 변경. "📂 불러오기 / 💾 저장 / 🗑" 형태. 삭제 버튼은 시각적으로 약하게(회색) 처리하여 위계를 구분합니다.

2. 사이클 바와 테이블의 관계 명확화

현행은 테이블과 사이클 바가 물리적으로 분리되어 있어, "어떤 행의 바인지" 연결이 약합니다. 멀티 경로 시 바는 대표 채널 1개만 표시하는데, 어떤 경로의 바인지 알 수 없습니다.

Variant A (인라인): 각 테이블 행 아래에 서브 행으로 타임라인을 직접 삽입. 테이블 행과 바의 1:1 관계가 명확. 단점: 테이블 높이 증가.

Variant B (분리): 현행 구조를 유지하되 좌측에 행 라벨 추가. 테이블과 바의 행 순서가 동일함을 시각적으로 보장. 단점: 동기화 필요.

3. 사이클 열(col4)의 시각적 강조

"사이클" 열은 CycleNo의 single source of truth이지만, 현행에서는 다른 열과 동일한 스타일. 사용자가 바에서 선택한 결과가 이 열에 기록된다는 연결을 인지하기 어렵습니다.

개선: 사이클 열 텍스트를 Navy bold로 강조. 바에서 선택 시 해당 셀에 짧은 하이라이트 애니메이션 추가 (PyQt6: QPropertyAnimation으로 배경색 전환).

4. 모드 열(col6)의 자동 표시

모드(General/Sweep)는 Phase 0에서 자동 판별되지만, 현행에서는 빈 셀로 남아있어 사용자에게 정보가 전달되지 않습니다.

개선: Phase 0 완료 후 "Gen" / "Swp" 약어를 자동 기입. 회색 글씨로 "자동 판별됨"을 시각적으로 표현. Sweep 모드일 때 주황 강조로 주의 유도.

5. 하단 상태바 정보

현행에는 입력 상태를 요약하는 정보가 없습니다. 경로가 몇 개 입력되었는지, 채널 수는 몇 개인지, 사이클 바가 어떤 모드인지 한눈에 알 수 없습니다.

개선: GroupBox 하단에 상태 텍스트 추가: "경로 3개 ⏐ 채널 3개 ⏐ 사이클 바: 논리 모드". 현재 활성 행 정보도 표시.

6. PyQt6 구현 참고

Variant A (인라인 바): QTableWidget의 setSpan()으로 서브 행 병합. 또는 setIndexWidget()로 CycleTimelineBar 미니 버전을 셀 안에 삽입. setRowHeight()로 서브 행 높이 조절.
상태바: QLabel + QHBoxLayout, GroupBox 하단에 배치.
모드 열: _autofill_table_empty_cells() 또는 Phase 0 완료 콜백에서 자동 기입.
사이클 열 강조: QItemDelegate로 폰트 색상/굵기 오버라이드.
