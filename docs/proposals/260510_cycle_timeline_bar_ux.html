0현재 상태 — 무엇이 좋고 무엇이 부족한가

2026-04-12 정비 이후 안정화된 위젯. 패턴 블록·다중 행·Shift/Ctrl·우클릭·눈금까지 갖춘 상태에서 시각·인터랙션의 6가지 빈틈을 보강한다.

기존 강점 (유지)
  • 패턴별 색상 (PALETTE 10색 · 18+ classified 카테고리)
  • 행별 독립 선택 + 다중 채널 비교
  • Shift/Ctrl/Drag · 우클릭 (이 블록 / 같은 패턴 / 전체)
  • 반사이클 0.3 가중 축소 + _MIN_BLOCK_PX=8 가시성 보장
  • 호버 툴팁 + 동적 눈금 (5/10/50/100/500)
남은 빈틈 (이번 개선)
  • 1. 큰 블록도 호버해야 카테고리 식별 — 인지부하
  • 2. 범례·필터 없음 — 색상-카테고리 학습 비용
  • 3. 6000+ 사이클 시 줌 부재 — 미세 패턴 분간 불가
  • 4. RPT/DCIR/GITT 진단 사이클이 일반 사이클에 묻힘
  • 5. _capacity_trend 변수만 있고 그리지 않음
  • 6. 선택 시 평균/fade%/시간 등 즉시 통계 부재

1인라인 라벨 + 카테고리 칩 범례 NEW

큰 블록(≥40px)에는 패턴 약어를 직접 그린다. 칩 클릭으로 해당 카테고리만 강조 (단독 모드)·Alt+클릭으로 토글. 패턴 학습 곡선이 사라지고, 한 번 보면 분포가 즉시 인지된다.

BEFORE

AFTER

PyQt6 구현 포인트

  • 라벨 그리기: paintEvent 블록 루프 안에서 bw >= 40 and pat['name']일 때 painter.drawText(QRect(x1,y,bw,h), Qt.AlignmentFlag.AlignCenter, abbrev)
  • 칩 범례: 위젯 외부 QWidget로 분리 (CycleLegendChips) → filterChanged(set[str]) 시그널 → CycleTimelineBar.set_filter(cats)
  • 줌: self._view = (start, end) 멤버 추가, _build_block_layout에 view 인자. wheelEvent로 줌 인/아웃, 미니맵은 별도 자식 위젯
  • 마커: classified 카테고리가 {'RPT','DCIR','GITT','GITT(full)','GITT(simplified)'}인 블록을 위쪽 16px 영역에 그림. setMinimumHeight를 16px 늘림
  • 용량 오버레이: 기존 self._capacity_trend 사용. QPainterPath로 점선 (DashLine, width=1.2, alpha=140)
  • 통계 카드: selectionChanged 시그널을 받아 부모 컨테이너에서 계산·렌더 (위젯 자체는 변경 최소화)
  • 정렬 동기화: set_multi_blocks(rows, sync_max=True) 옵션 추가. True면 total_cycles 대신 global_max 기준으로 layout 계산
