Cycle / Profile 탭 — UX 분석 노트

1

DCIR 옵션 라벨 과부하

현행: 라디오 버튼 라벨에 "PNE 설비 DCIR (SOC100 10s 방전 Pulse)" 같은 긴 텍스트가 한 줄에 나열. 3번째 옵션은 2줄 라벨. 시각적 스캔이 어렵고 옵션 간 차이를 빠르게 파악하기 힘듬.

개선: 카드형 선택 UI — 짧은 이름(설비 DCIR / 10s Pulse / MK DCIR) + 설명 분리 + 뱃지 태그. 선택된 카드에 파란 테두리 강조. 수직 스캔 시 이름만 읽으면 구분 가능.

PyQt6 구현: QFrame + QVBoxLayout + paintEvent에서 선택 시 border 색상 변경. 또는 QButtonGroup + custom QRadioButton delegate.
2

그래프 옵션 입력 필드 문맥 부재

현행: "Y축 최대 1.10 Y축 최소 0.65"처럼 라벨과 입력이 수평 나열. 어떤 값이 어떤 필드인지 시선이 교차됨. 단위 표기 없음. "DCIR scale 늘리기 (x ?배)"는 라벨 자체가 질문형이라 혼란.

개선: "최소 ~ 최대" 범위 패턴 + 단위 표기(V, 배). 연한 배경의 필드 그룹으로 시각적 영역화. "0=auto" 힌트 추가.

PyQt6 구현: QFrame(배경색 #FAFAFA) 안에 QGridLayout. QLabel + QLineEdit 조합. placeholder 또는 suffix QLabel로 단위 표시.
3

실행 버튼 위계 미흡

현행: "Cycle 분석"과 "초기화"가 같은 크기. Primary 액션과 보조 액션의 시각적 차이가 약함. 초기화는 disabled 상태인데 왜 비활성인지 피드백 없음.

개선: Primary 버튼 flex:2 + 큰 폰트 + ▶ 아이콘. 초기화는 flex:1 + 회색 텍스트. 분석 실행 후에만 초기화 활성화(상태 기반).

PyQt6 구현: QPushButton#primaryBtn에 QSS 적용. setSizePolicy(Expanding). 초기화는 setEnabled(False) + 분석 완료 시그널로 활성화.
4

라디오 버튼 → Segmented Control

현행: "개별 탭 / 통합 탭", "데이터: 사이클/충전/방전", "연속성: 오버레이/이어서" 등 2~3개 짧은 선택지가 라디오 버튼으로 나열. 원형 라디오가 반복되어 시각적 잡음 많음.

개선: Segmented Control(토글 버튼 그룹)로 교체. 선택지가 명확하고 컴팩트. 선택 상태가 배경색으로 즉시 인지됨.

PyQt6 구현: QButtonGroup + QPushButton(setCheckable). QSS로 :checked 상태 스타일링. 또는 커스텀 QSegmentedControl 위젯.
5

필드 그룹핑으로 영역 인지성 향상

현행: GroupBox 내부의 라디오/입력이 같은 배경(#F7F7F7)에 나열. 데이터 범위 안의 "데이터/연속성/X축" 3개 설정군이 시각적으로 분리되지 않음.

개선: 관련 필드를 연한 배경(#FAFAFA) + 1px 테두리의 서브프레임으로 묶어 시각적 계층화. 같은 맥락의 설정은 한 프레임 안에 모아서 "이 영역이 하나의 설정 단위"임을 전달.

PyQt6 구현: QFrame(background: #FAFAFA; border: 1px solid #E8E8E8) + QVBoxLayout/QGridLayout 내부.
6

경로 테이블 — 행 수 조절 + 버튼 재배치

현행: 5행 고정이지만 대부분 1~2행만 사용. 빈 행 3~4개가 공간 낭비. 우측 아이콘 3개는 텍스트 없어 기능 추측 필요.

개선: 기본 3행(입력 시 자동 확장). 버튼을 체크박스 행 우측으로 이동(아이콘+텍스트). 사이클/Raw 열 헤더를 Navy bold로 강조.

PyQt6 구현: setRowCount(3) 기본, cellChanged 시그널에서 마지막 행 입력 감지 → insertRow(). 헤더 QSS로 컬럼별 폰트 스타일 차별화.
