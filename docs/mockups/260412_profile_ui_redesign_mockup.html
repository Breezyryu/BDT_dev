UX 문제점 분석 및 개선 제안

1. 정보 계층 부재 — 모든 옵션이 동일한 무게

현행 UI는 6개 입력 필드, 3개 라디오 그룹, 3개 체크박스가 모두 같은 시각적 무게로 나열되어 있어, 사용자가 "어떤 설정이 핵심이고 어떤 것이 부가인지" 파악하는 데 인지 비용이 높습니다.

Before 전압 Y축 하한 / 상한 / 간격 / Smooth / 컷오프 / dQdV 축늘리기가 2열 그리드에 무분류 나열. 처음 쓰는 사용자는 어떤 값부터 봐야 하는지 모름.
→
After 접을 수 있는 섹션으로 분리: "데이터 범위"(핵심)와 "그래프 설정"(부가). 자주 바꾸는 옵션은 펼쳐두고, 한번 정하면 안 바꾸는 설정은 접어둠.

2. 라디오 버튼 그룹 경계 모호

"데이터: 사이클/충전/방전"과 "연속성: 오버레이/이어서"가 한 줄에 텍스트 라벨만으로 구분되어, 어디까지가 한 그룹인지 시각적으로 불명확합니다. 두 번째 줄은 라디오(X축)와 체크박스(Rest, CV, 루프)가 혼재해서 더 혼란스럽습니다.

Before 데이터: ○사이클 ○충전 ○방전   연속성: ○오버레이 ●이어서
라벨 텍스트만으로 그룹 구분. 시선이 빠르게 스캔할 때 그룹 경계를 놓치기 쉬움.
→
After Segmented Control (세그먼트 버튼)로 각 그룹을 물리적 박스로 분리. 그룹 위에 미니 라벨("데이터", "연속성", "X축")을 배치하여 소속을 명확히 함.

3. 주요 액션 버튼의 시각적 위계 부족

"프로필 분석", "DCIR", "탭 초기화" 버튼이 모두 같은 크기·스타일입니다. 가장 많이 누르는 "프로필 분석"과 파괴적 액션인 "초기화"가 동등하게 보이면, 실수 클릭 확률이 높아집니다.

Before [프로필 분석] [DCIR] [탭 초기화]
3개 버튼 동일 크기·색상. Primary action 구분 없음.
→
After 프로필 분석: Navy filled + 2x 너비 (Primary)
DCIR: White bordered (Secondary)
초기화: Light gray (Tertiary/Danger)

4. 입력 필드의 맥락 정보 부족

"Smooth (0 이면 자동)" — 괄호 안 힌트가 라벨에 붙어 있어 읽기 불편합니다. "컷오프 (전류,전압)"도 무엇을 넣어야 하는지 직관적이지 않습니다. 단위 표시도 없습니다.

Before 라벨: Smooth (0 이면 자동)
힌트가 라벨과 같은 줄에 괄호로 들어가 있어 라벨 자체가 길어짐. 단위(V, C-rate) 표시 없음.
→
After 라벨: Smoothing (짧은 대문자)
플레이스홀더: 0 = 자동
하단 힌트: 0 입력 시 자동 결정
단위: 입력 필드 우측에 V, C 등 명시

5. 사이클 타임라인 바의 정보 밀도 개선

현행 타임라인은 색상 블록만 표시하고 카테고리 라벨이 없어, 어떤 블록이 Aging이고 RPT인지 바 위에서 직접 읽을 수 없습니다. TC/상세 토글 버튼도 작아서 발견하기 어렵습니다.

Before 색상만 있는 바 + 우측 끝 [TC] [▶] 소형 버튼
블록 의미를 알려면 ▶ 팝업을 열어야 함
→
After 각 블록에 카테고리 라벨 인라인 표시 (Aging, RPT 등)
상단에 논리/TC/상세 세그먼트 토글로 모드 전환
선택된 블록은 노란 테두리로 하이라이트

6. 전압 범위 입력의 물리적 직관성

"Y축 하한"과 "Y축 상한"이 별도 줄에 있어 범위 개념이 시각적으로 드러나지 않습니다.

Before 전압 Y축 하한 [2.5]   전압 Y축 상한 [4.7]
두 입력이 물리적으로 떨어져 있어 "범위" 개념 약함
→
After [2.5] ~ [4.7] V — 한 줄에 하한~상한을 인라인 레인지로 묶고 단위(V) 명시. 라벨은 상단에 "전압 Y축 범위" 하나로 통합.
