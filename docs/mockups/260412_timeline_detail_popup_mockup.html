UX 분석: 상세 팝업 개선 포인트

1. 플랫 텍스트 → 구조화된 카드 그리드

현행은 블록 정보가 "■ → ■ → ■ → ..." 형태의 단일 줄 텍스트로, 블록 수가 많으면 가로 스크롤이 필요합니다. 블록별 색상·범위·카운트를 한눈에 비교하기 어렵습니다.

Before ■ 가속수명 [3-101] 99cy → ■ RPT [102-103] 2cy → ...
한 줄에 모든 블록이 "→"로 연결. 블록이 7개 이상이면 줄 넘침.
→
After 카드 그리드: 블록별 독립 카드로 색상 스워치 + 이름 + 범위 + 카운트 구조화. + 패턴 흐름 다이어그램: 반복 구조를 압축 표현.

2. RPT 숫자 나열 → 위치 시각화 + 요약

현행 RPT 마커는 숫자 목록("102, 103, 203, 204")만 표시하여, 타임라인 상 어디에 위치하는지 직관적으로 알 수 없습니다.

Before ▼ RPT: 102, 103, 203, 204 (4회)
숫자만 나열. RPT 간격이 일정한지, 어디쯤인지 한눈에 파악 불가.
→
After 뱃지 + 미니 타임라인 마커: RPT 위치를 바 위에 삼각형으로 표시.
요약 패널: "4회, ~100cy 간격, 마지막 Cy 204"로 핵심 통계 제공.

3. 텍스트 게이지 → 그래픽 프로그레스 바

현행 채널 진행은 ████████░░░ 텍스트 게이지로, 비율 차이가 작을 때 (예: 308 vs 305) 시각적으로 구분이 거의 불가능합니다.

Before ████████████████ CH008 308cy (기준) / ███████████████░ CH014 305cy
15칸 텍스트 블록 — 308:305 차이를 시각으로 식별 불가.
→
After 그래픽 프로그레스 바: 실제 비율 기반 너비. 그라데이션으로 시각적 풍성함 추가.
채널별 독립 행 + 기준 태그 + 하단 통계(최대-최소 차이, 평균).

4. 4개 정보의 혼합 나열 → 탭 분리

현행은 블록·RPT·선택·채널 4가지 정보가 하나의 텍스트 블록에 순서대로 나열됩니다. 원하는 정보를 찾으려면 전체를 읽어야 합니다.

Before 4가지 정보가 줄 단위로 순차 나열. 채널 진행만 보고 싶어도 블록 라벨 줄을 먼저 지나야 함. 정보량이 많으면 패널이 길어짐.
→
After 미니 탭(블록 구성 / RPT 마커 / 채널 진행 / 선택 정보)으로 분리. 한 번에 하나의 맥락에 집중. 패널 높이 일정하게 유지.

5. 패턴 반복 구조의 압축 표현

배터리 수명 시험은 "가속수명 99cy + RPT 2cy"가 반복되는 규칙적 구조입니다. 현행은 이 반복을 풀어서 나열하므로 전체 시험 설계 의도가 즉시 드러나지 않습니다.

Before ■가속 → ■RPT → ■가속 → ■RPT → ■가속 → ■Rss
반복이 풀어져 있어 "3회 반복" 구조를 사용자가 직접 추론해야 함.
→
After 패턴 흐름 다이어그램: 초기 → [가속99cy → RPT2cy] ×3 → Rss5cy
반복 구조를 "×3 반복" 노드로 압축. 시험 설계 의도가 한눈에.

6. PyQt6 구현 고려사항

현행 QTextDocument HTML 렌더링 → QWidget 기반 레이아웃 전환 시 고려할 점:

블록 그리드: QGridLayout + QFrame 카드. 색상 스워치는 QLabel + QSS background.
미니 탭: QStackedWidget + QPushButton 그룹 (QButtonGroup exclusive).
채널 게이지: QProgressBar 커스텀 스타일 또는 paintEvent 직접 그리기.
RPT 마커: 미니 타임라인은 CycleTimelineBar의 축소 복제 — paintEvent에서 삼각형 마커 추가.
높이 관리: 탭 전환 시 높이 고정 (현행의 동적 높이 변화 문제 해소).
