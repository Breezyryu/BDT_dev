멀티 행 상세 팝업 — UX 분석

1. 행 라벨 추가 — "어떤 바가 어떤 시험인지"

현행 바에는 행 라벨이 없어서, 3행 이상일 때 어떤 바가 수명시험이고 어떤 바가 GITT인지 구분할 수 없습니다.

Before행 구분 없이 색상만 의존. 활성 행의 테두리 차이(0.5px vs 1.0px)는 미미함.
→
After좌측에 경로 라벨(수명-45℃, GITT-23℃ 등) 추가. 행 클릭 시 활성 전환이 직관적.

2. 경로 선택 필(Pill) — 상세 패널과 바의 연결

현행은 상세 정보가 "경로 1 / 경로 2 / 경로 3" 헤더로 전부 나열됩니다. 원하는 경로만 보려면 스크롤해야 합니다.

Before모든 경로의 4가지 정보가 순차 나열 → 3행이면 12줄 이상. 경로 간 전환은 스크롤.
→
After상세 패널 상단에 경로 필(pill) 토글. 한 번에 한 경로만 표시. + "전체 요약" 모드로 경로 간 비교.

3. "전체 요약" 모드 — 경로 간 비교

현행에는 경로 간 비교 뷰가 없습니다. 각 경로의 사이클 수, RPT 유무, 채널 수를 비교하려면 텍스트를 하나씩 읽어야 합니다.

Before경로별 정보가 분리되어 나열. 크로스 비교는 사용자의 멘탈 모델에 의존.
→
After오버뷰 카드: 각 경로를 카드로 요약 (사이클 수, 블록 수, RPT 수, 채널 수 + 미니 바).
크로스 테이블: 전체 선택 현황, 전체 RPT 분포, 전체 채널 현황을 표로 비교.

4. 바 ↔ 상세 패널 양방향 연동

현행은 바에서 행을 클릭하면 활성 행이 바뀌지만, 상세 텍스트는 전체가 나열되어 있어 어떤 행이 활성인지 연결감이 없습니다.

Before바 행 전환 → 상세 텍스트는 변화 없음 (모든 경로가 항상 표시). 활성 행 컨텍스트 단절.
→
After바 행 클릭 → 상세 패널의 경로 필도 동기화. 상세 패널의 필 클릭 → 바 활성 행도 동기화. 양방향 바인딩.

5. 높이 안정성

현행은 상세 텍스트가 모든 경로를 한 번에 표시하므로, 경로 수가 늘면 패널 높이가 비례해서 커집니다. 3행 × 4정보 = 최소 12줄 이상.

Before경로 수 × 정보 수 = 패널 높이 선형 증가. 3행이면 ~200px+.
→
After한 번에 1경로 × 1탭만 표시 → 높이 일정 (~140px). 경로/탭 전환은 클릭. QScrollArea 불필요.

6. PyQt6 구현 매핑

경로 필(pill): QHBoxLayout + QPushButton(checkable, exclusive) → QButtonGroup
바 ↔ 필 연동: 바의 activeRowChanged(int) 시그널 → 필 동기화 슬롯, 반대도 동일
전체 요약 카드: QGridLayout + QFrame 카드, 클릭 시 해당 행으로 전환
크로스 테이블: QTableWidget (compact, read-only) 또는 QLabel HTML
높이 관리: QStackedWidget으로 경로별 콘텐츠 전환 → 높이 고정
